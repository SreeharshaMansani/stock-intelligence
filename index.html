Pipeline

How it works

Eight sequential phases run every weekday morning. Each phase feeds into the next — from raw RSS feeds and Yahoo Finance prices to a fully rendered HTML email in your inbox.

PHASE 01
Build Stock Universe
Reads your watchlist and sector exposure mappings from two tabs of a Google Sheet via the Sheets API.
Google Sheets APIService Account
PHASE 02
Fetch Macro Context
Pulls Nifty 50, Sensex, S&P 500, Nasdaq, Brent Crude, Gold, USD/INR, and US 10Y yield from Yahoo Finance. Also fetches India-focused macro headlines from Google News RSS.
Yahoo FinanceGoogle News RSS
PHASE 03
Build Exposure Map
Fetches news for every sector / thematic exposure term linked to your stocks — commodities, regulators, peers, currencies — in parallel batches.
Google News RSSBatch Fetching
PHASE 04
Per-Stock Processing
For each stock: fetch RSS news → filter by alias & date → deduplicate → score sentiment → build summary bundle → run T5 summariser → fetch Yahoo price data. All in parallel.
Google NewsT5 SummariserYahoo FinanceSQLite
PHASE 05
Read Recent Rows
Queries SQLite for the last 1–3 days of stock data (3 days on Mondays to cover the weekend gap).
SQLitebetter-sqlite3
PHASE 06
Build Gemini Prompt
Assembles all stock data, macro context, and exposure news into a structured prompt with strict output format rules. Falls back to a degraded prompt if the sheet or data pipeline fails.
Prompt Engineering
PHASE 07
Generate with Gemini
Calls Gemini via Google AI Studio (API key) or Vertex AI (service account). Falls back to a rule-based local generator if both are unavailable.
Gemini 2.5 FlashVertex AIFallback Generator
PHASE 08
Render HTML & Send Email
Converts the Markdown report to a styled HTML email. Archives it locally. Sends via Resend HTTP API (port 443, no SMTP blocks).
HTML TemplateResend APILocal Archive

Four signal types

Every stock in your report gets one of these directional signals, determined by a rule-based rubric applied to sentiment score, price momentum, and distance from the 52-week high.

BUY
Positive sentiment AND price not significantly down AND >5% below 52w high
HOLD
Neutral/positive sentiment AND price near 52w high — keep positions, no new entry
WAIT
Insufficient news, mixed signals, or price data unavailable
SELL
Negative sentiment AND price showing weakness

Prerequisites

Get your credentials

You need four free accounts. None of them require a credit card.

01 / GOOGLE SHEETS
Service Account
Gives the pipeline read-only access to your watchlist sheet.
  • Create a GCP project, enable Sheets API
  • Create a service account → download JSON key
  • Rename to google-service-account.json
  • Share your sheet with the service account email as Viewer
02 / GEMINI
AI Studio Key
Free-tier API key for Gemini 2.5 Flash report generation.
  • Go to aistudio.google.com
  • Click Create API Key
  • Copy the AIzaSy… key
03 / RESEND
Email API Key
Sends your HTML report via HTTPS (no SMTP port blocks).
  • Sign up at resend.com
  • Create API key with Sending access
  • Free tier: 3,000 emails/month
04 / GOOGLE SHEET
Watchlist Sheet
Two tabs define everything the pipeline tracks.
  • Stocks tab: ticker, query, aliases, sector_summary
  • exposure tab: ticker, exposure_term pairs
  • Copy the Spreadsheet ID from the URL

Local Development

Run it locally

terminal
# Install dependencies
npm install

# Dry-run: full pipeline, no email sent
node index.js --now --dry-run

# Full run: pipeline + email dispatch
node index.js --now

# Start dashboard server + built-in scheduler
npm run dashboard
⚠️ The dry-run flag skips email dispatch and saves the HTML report to ./reports/ instead. Open it in a browser to preview exactly what would have been sent.
Deployment

Deploy for free in 6 steps

01
Push to GitHub
Create a private repo. The .gitignore ensures secrets never leave your machine.
02
Create Render Service
New → Web Service → connect your repo. Build: npm install. Start: npm run dashboard. Instance: Free.
render.com →
03
Add Env Vars
Add every key from the table above under Environment Variables in the Render dashboard.
04
Upload Secret File
Under Secret Files, add google-service-account.json and paste your service account key JSON.
05
Deploy
Click Create Web Service. Render builds and deploys automatically. Note your .onrender.com URL.
06
Schedule with GitHub Actions
Create .github/workflows/daily-run.yml to run on a cron schedule, pinging your Render URL. Set to 7:00 AM IST, Mon–Fri.
GitHub Actions →
💾 Render's free tier has an ephemeral filesystem — the SQLite DB and report archives reset on redeploy. This doesn't affect daily delivery. For permanent history, point src/db.js at a free Supabase PostgreSQL database instead.
